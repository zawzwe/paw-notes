# 宠物声音情感翻译网站 - 用户旅程地图

## 1. 文档概述

- 本项目旨在为猫狗主人提供一个在线平台，通过录入宠物声音，利用大模型分析并转换为人类语言，帮助主人理解宠物情绪
- 文档以手机端适配为重点，覆盖实时录音与上传音频两种使用场景，支持中英文输出
- 本用户旅程地图基于预设的典型用户角色，梳理了从认知到推荐的完整体验流程，并提炼痛点与机会点

## 2. 用户角色画像（Persona）

- **姓名**：Jessie
- **年龄**：30岁
- **职业**：远程办公的电商运营，时间灵活但碎片化
- **宠物**：养了一只猫（小咪）和一只狗（旺财），两者都偶尔出现异常叫声
- **技术习惯**：频繁使用手机，习惯在TikTok上浏览萌宠内容，喜欢尝试新奇的数字工具
- **核心目标**：实时读懂宠物情绪，减少因误判叫声含义导致的焦虑；记录宠物的“声音日记”并与朋友分享
- **痛点**：现有动物行为知识获取渠道分散，缺乏科学、即时、有说服力的翻译工具；担心录音隐私安全且过程太慢

## 3. 核心使用场景

- **场景A：实时突发分析**：在家听到旺财突然狂吠，Jessie立即打开App录音，几秒后听到“有人靠近门口，我有点紧张”的人声翻译
- **场景B：通勤时上传音频**：Jessie在散步时录下小咪连续的喵喵声，回家后上传至平台，获得“我想吃饭，但碗里还有”的长句分析
- **场景C：睡前回顾与分享**：Jessie查看一天中所有分析记录，将“旺财开心地迎接”片段转为语音收藏并一键分享到朋友圈

## 4. 用户旅程阶段

| 阶段 | 用户行为与情绪 | 关键触点 |
| :--- | :--- | :--- |
| **认知 (Awareness)** | 在TikTok刷到萌宠翻译广告, 产生好奇；查看评论区，发现真实用户晒出宠物对话效果 | TikTok信息流、广告创意、评论区 |
| **考虑 (Consideration)** | 访问官网首页，阅读“科学训练”说明；浏览用户评价和示范视频；体验免费示例音频（快速分析狗叫） | 网站首页、示例页、FAQ |
| **引导入门 (Onboarding)** | 注册账号，选择宠物类型（猫/狗）并录制初始化语音样本；看到欢迎动画和简单使用指南 | 注册页、宠物设置页、引导弹窗 |
| **使用 (Usage)** | 实时录音或上传文件，等待分析结果（约5-15秒）；听到TTS生成的人类语音，并查看情绪标签结果；保存记录并重复使用 | 录音/上传按钮、进度条、分析结果页、历史记录 |
| **倡导 (Advocacy)** | 将有趣的翻译结果截图或生成短视频分享至社交媒体；主动向养宠群友推荐该平台 | 分享按钮、社交平台API、口碑社区 |

## 5. 痛点与机会点

- **痛点1：录音质量不稳定导致分析失败**
  - 用户可能在嘈杂环境录音，背景噪音干扰模型识别
  - 机会点：提供录音前噪音检测提示及简易降噪建议；允许多次重置录音

- **痛点2：分析耗时超出预期**
  - 用户期待秒级返回，但模型处理加上TTS转换可能需要10-15秒，容易产生焦躁情绪
  - 机会点：加入进度动画或倒计时预估，并允许用户滑动查看历史分析结果作为缓冲

- **痛点3：翻译结果缺乏可信度**
  - 用户难以判断“旺财说想吃饭”是否准确，可能认为只是随机文本
  - 机会点：显示置信度百分比及分析依据摘要（如频率、时长特征），增加解释性

- **痛点4：隐私顾虑与数据管理**
  - 用户担心录音被滥用或上传后不易删除，不符合海外合规预期（如GDPR/CCPA）
  - 机会点：在首次录音时明确展示隐私政策、数据删除流程，并提供本地加密存储选项

- **痛点5：缺乏社交互动与反复使用的激励**
  - 用户完成一次翻译后可能忘记再次打开，使用粘性不足
  - 机会点：设置宠物“声音日记”打卡任务，并提供勋章或社区排名功能

## 6. 服务蓝图要点

- **用户行为**：
  - 访问网站、注册账户、配置宠物资料、执行录音/上传、查看结果、分享
  - 高频行为集中在移动端小屏幕场景，需确保按钮触控友好、页面加载快

- **前台交互**：
  - 录音按钮采用大圆型麦克风设计，带实时波形动画；上传支持从相册或文件管理选取音频
  - 分析结果页面采用卡片布局，顶部显示主情绪标签，下方为TTS播放器和详细解读文本

- **后台流程**：
  - 使用Next.js在Vercel上构建前端，Node.js后端负责接收音频文件，调用第三方语音识别API
  - 利用Supabase管理用户数据、宠物档案、录音文件及历史记录；过期数据自动清理符合规要求

- **支持系统**：
  - 第三方API：语音识别模型（如Google Speech-to-Text或Whisper）、情感分析模型、TTS语音合成
  - 监控系统：对处理时长（目标＜15秒）和错误率（目标＜5%）设置告警；TikTok广告投放后需支持高并发（预计日均PV 5000+）

## 7. 结论与建议

- **优先建设核心识别准确度与速度**：确保不秒级但需稳定在10-15秒内，尽量减少因模型精度不足导致的无效分析；可先在内部测试常用猫狗叫声类型
- **强化新用户引导与信任感**：通过样本音频展示真实案例，并在首次使用时清晰解释“科学训练”含义及隐私政策，降低疑虑
- **规划社交分享与内容运营**：内置一键生成带品牌水印的短视频或图片，辅助TikTok投放；同时建立声音社区，鼓励用户分享有趣翻译并举办话题活动
- **预留性能增量方案**：采用Vercel + Supabase的无服务器架构，通过缓存热门分析结果、异步任务队列应对短期高峰流量；后续可引入CDN加速音频资源分发
