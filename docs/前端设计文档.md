# 前端设计文档：PetVoice 宠物声音翻译助手

## 1. 技术栈推荐

| 类别 | 技术 | 说明 |
|------|------|------|
| 框架 | Next.js 14+ (App Router) | 基于 React，服务端渲染 + API 路由，部署 Vercel |
| 类型系统 | TypeScript | 严格模式，保障代码质量 |
| 样式方案 | Tailwind CSS + clsx/twMerge | 移动端优先，快速响应式 |
| 状态管理 | Zustand + React Query | 轻量级全局状态 + 服务端缓存/请求 |
| 表单与数据 | React Hook Form + Zod | 录音/上传表单验证 |
| UI 组件 | Headless UI + shadcn/ui | 无障碍，可自定义主题 |
| 语音处理 | RecordRTC / MediaRecorder API | 浏览器端录音，支持实时流或音频文件 |
| 音频可视化 | wavesurfer.js | 显示录音波形（可选） |
| 第三方 SDK | 第三方大模型 API（如 Whisper + GPT）、TTS API | 通过后端封装，前端仅发起请求 |

## 2. 页面与路由设计

采用移动端优先，页面结构扁平：

| 路由 | 页面名称 | 功能描述 |
|------|----------|----------|
| `/` | 首页 / 录音页面 | 核心功能：选择动物（猫/狗）、录音或上传、显示分析结果、播放 TTS |
| `/history` | 历史记录 | 展示用户所有分析记录（日期、动物、情绪摘要、音频回放） |
| `/auth/login` | 登录/注册 | 邮箱+密码登录（Supabase Auth） |
| `/auth/callback` | 认证回调 | 处理 OAuth 或 Magic Link |
| `/profile` | 个人设置 | 语言偏好（中文/英文）、账户管理、隐私设置 |

### 路由设计细节
- 所有页面顶部需有简洁导航栏：Logo、历史记录入口、用户菜单（登录后）
- `/` 页面为默认入口，未登录用户可体验但无法保存历史（仅展示一次结果）
- 录音/上传流程：选择动物 → 录音或上传 → 显示分析中 → 展示情绪描述 + TTS 播放
- 历史记录页面采用无限滚动加载（基于 Supabase 分页）

## 4. 状态管理方案

### 4.1 全局状态（Zustand）

```typescript
// store/appStore.ts
interface AppState {
  selectedAnimal: 'dog' | 'cat' | null;
  currentAudioBlob: Blob | null;
  analysisResult: AnalysisResult | null; // 包含情绪文本、置信度、TTS URL
  isAnalyzing: boolean;
  // 操作
  setAnimal: (type: 'dog' | 'cat') => void;
  setAudio: (blob: Blob) => void;
  setResult: (result: AnalysisResult | null) => void;
  reset: () => void;
}
```

### 4.2 服务端状态（React Query / TanStack Query）

- **缓存策略**：`staleTime: 5分钟`（历史列表），`gcTime: 10分钟`
- **关键 Query**：
  - `['history', { page, limit }]` → 分页历史记录
  - `['analysis', { audioId }]` → 分析结果详情（用于历史页回看）
- **Mutation**：
  - `useAnalyzeAudio` → 上传音频到后端，返回分析结果（触发 loading 状态）

### 4.3 认证状态

- 使用 Supabase Auth 的 `onAuthStateChange`，通过 Context 或 Zustand 存储 `user` 和 `session`
- 未登录用户限制：无法查看历史、无法保存记录（前端阻止页面跳转）

## 5. 响应式与交互设计

### 5.1 移动端优先
- 基准设备：iPhone 12/13 尺寸 (390×844)，断点：`sm: 640px`，`md: 768px`（极少使用）
- 所有内容基于 `max-w-md mx-auto` 居中，边距 `p-4`
- 按钮大小 `min-h-[48px]`，点击区域≥44px
- 底部固定导航栏（渐隐顶部导航），首页显示录音主区域

### 5.2 录音交互流程
```
[选择动物] → [录音按钮（大圆）] → 录音中（倒计时+波形） → 停止 → 
[预览播放] → [分析按钮] → loading动画 → [结果卡片 + TTS播放]
```
- 录音按钮提供振动反馈（Haptic，仅支持设备）
- 分析过程中用骨架屏 + 进度条表示
- 结果卡片包含：情绪图标、文字描述（如“开心地摇尾巴”）、置信度百分比、TTS 播放按钮（可切换中文/英文）

### 5.3 上传交互
- 代替录音按钮下方提供“从文件上传”链接
- 上传后自动进入分析流程，不显示预览
- 支持 `audio/*` 格式（mp3, wav, m4a），最大 10MB

### 5.4 无障碍 (A11y)
- 所有交互元素提供 `aria-label`
- 录音状态变化通过 `role="alert"` 播报
- 颜色对比度符合 WCAG AA

## 6. 性能优化方案

### 6.1 构建与部署
- 使用 Next.js 静态生成 (SSG) 首页和公共内容（如关于页面）
- 动态路由 `/history` 和 `/profile` 采用客户端渲染 (CSR) + 骨架屏
- 图片（图标、动物图片）使用 `next/image` 优化

### 6.2 网络与资源
- 音频文件直接上传到 Supabase Storage（或后端转发），避免占用 Vercel 带宽
- API 请求使用 HTTP/2，前端对同一个端点做 `keep-alive`
- 关键 JS 预加载（如 `Recorder` 组件对应的 `RecordRTC` 库在需要时才加载）

### 6.3 状态与渲染
- 避免不必要的 re-render：使用 `React.memo` 包裹结果卡片、历史列表项
- 使用 `useDeferredValue` 处理大量历史列表的模糊搜索（未来功能）
- 录音组件在非活动状态时释放 MediaStream

### 6.4 加载体验
- 录音功能首次使用时提示授予麦克风权限
- 分析结果采用流式渲染：先返回情绪文本，再一边加载 TTS 音频（如果后端支持流式 TTS）
- 历史页面使用 `IntersectionObserver` 实现无限滚动，配合预取 `next page`

### 6.5 并发与缓存
- React Query 自动去重相同请求
- 使用 `staleWhileRevalidate` 策略，分析结果可缓存 10 秒（避免重复分析同一音频）
- 后端返回 CDN 链接的 TTS 音频（Supabase Storage 自带 CDN）

## 7. 开发建议

### 7.1 开发环境搭建
- 使用 `create-next-app` 初始化，开启 TypeScript + App Router + Tailwind
- 安装必要依赖：`zustand` `@tanstack/react-query` `@supabase/supabase-js` `@supabase/ssr` `react-hook-form` `zod` `recordrtc` `wavesurfer.js`
- 配置 Supabase 环境变量（`NEXT_PUBLIC_SUPABASE_URL` `NEXT_PUBLIC_SUPABASE_ANON_KEY`）

### 7.2 后端约定
- 使用 Next.js API Routes（`/api/analyze` `/api/upload`）作为 BFF，转发到第三方模型 API
- 音频上传限制：`bodyParser: false`，手动解析 `multipart/form-data`
- 返回格式统一：
```json
{
  "animal": "dog",
  "emotion": "happy",
  "description": "小狗正在开心地摇尾巴，可能想邀请你玩耍",
  "confidence": 0.92,
  "ttsUrl": "https://xxx.supabase.co/storage/v1/object/public/tts/xxx.mp3"
}
```

### 7.3 数据库设计（Supabase）
- `profiles` 表：`id (uuid)` `preferred_language` `created_at`
- `analysis_history` 表：`id` `user_id` `animal` `audio_url` `result_jsonb` `created_at`
- 启用 Row Level Security (RLS)：用户只能查看自己的记录

### 7.4 测试与质量
- 使用 Vitest + Testing Library 编写单元测试（重点：`useRecording` hook、组件逻辑）
- 录音组件可在 Storybook 中交互测试
- E2E 测试使用 Playwright，覆盖核心路径：录音 → 分析 → 播放 TTS

### 7.5 国际化 (i18n)
- 使用 `next-intl` 或 `i18next`，支持中英文
- 动态切换语言时，重新请求 TTS（后端返回对应语言版本）

### 7.6 发布与监控
- 部署 Vercel 后配置域名和 SSL
- 集成 Sentry 前端错误监控
- 在 Vercel Analytics 中跟踪用户流程（录音、上传成功率）

> 本设计文档专注于前端，后端 API 及大模型对接需同步规划。所有第三方 API Key 应存储在环境变量中，通过后端转发，避免前端暴露。
